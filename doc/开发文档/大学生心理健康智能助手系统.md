## 🧠 二、理论基础完善

### 2.1 理性情绪疗法(REBT)整合

基于参考材料[^6](%5B%E7%90%86%E6%80%A7%E6%83%85%E7%BB%AA%E7%96%97%E6%B3%95%5D\(https://baike.baidu.com/item/%E7%90%86%E6%80%A7%E6%83%85%E7%BB%AA%E7%96%97%E6%B3%95/635105\))，引入理性情绪疗法的ABCDE模型：

#### 2.1.1 ABCDE理论框架

-   **A (Activating Event)**：激发事件（用户遇到的问题）
    
-   **B (Belief)**：信念系统（对事件的认知和评价）
    
-   **C (Consequence)**：情绪和行为后果
    
-   **D (Disputation)**：驳斥（识别和挑战非理性信念）
    
-   **E (Effect)**：效果（建立新的理性信念和积极情绪）
    

#### 2.1.2 系统中的应用

```text
用户输入问题(A) → 识别信念模式(B) → 分析情绪后果(C) → 
智能驳斥非理性信念(D) → 提供理性替代方案(E)
```

### 2.2 情感计算理论支持

参考毛峡教授在情感计算和人机交互领域的研究成果 [^1](%5B%E6%AF%9B%E5%B3%A1%5D\(https://baike.baidu.com/item/%E6%AF%9B%E5%B3%A1/2484477\))：

-   **情感识别**：多模态情感分析（文本、语音、表情）
    
-   **情感建模**：情感状态量化和趋势分析
    
-   **情感交互**：基于情感状态的自适应交互策略

#### 3.1.2 Dify工作流详细设计

##### 🔄 **工作流1：情绪分析与干预工作流**

```yaml
name: 情绪分析与干预
trigger: 用户对话输入
steps:
  1. 情感分析:
      type: sentiment_analysis
      model: 自训练情感分类模型
      output: 情感类型、强度、关键词
      
  2. 信念识别:
      type: belief_extraction
      knowledge_base: REBT知识库
      output: 非理性信念识别
      
  3. 风险评估:
      type: risk_assessment
      rules: 预警规则库
      output: 风险等级、建议措施
      
  4. 干预策略生成:
      type: intervention_generation
      agents: [家庭矛盾Agent, 学业压力Agent, 考试焦虑Agent]
      output: 个性化干预方案
      
  5. 响应生成:
      type: response_generation
      template: 安抚+建议+资源推荐
      output: 用户回复
```

##### 🔄 **工作流2：人工转接工作流**

```yaml
name: 人工转接处理
trigger: 高风险识别或用户请求
steps:
  1. 风险确认:
      type: risk_confirmation
      conditions: 
        - 情感强度 > 8
        - 风险等级 = "高"
        - 用户明确请求人工帮助
      
  2. 专业匹配:
      type: professional_matching
      criteria: 问题类型、紧急程度、专业领域
      output: 匹配的心理咨询师/辅导员
      
  3. 信息整理:
      type: information_summary
      content: 对话历史、情绪分析、风险评估
      output: 专业人员参考资料
      
  4. 转接执行:
      type: handover_execution
      methods: [在线聊天, 电话联系, 预约面谈]
      output: 转接结果
```

### 4.2 Agent协作机制

#### 4.2.1 智能调度策略

```python
def agent_dispatcher(user_input, emotion_analysis, risk_level):
    """
    基于用户状态智能调度Agent
    """
    if risk_level == "高":
        return [危机干预Agent, 情绪安抚Agent]
    elif emotion_analysis["primary_emotion"] in ["抑郁", "焦虑"]:
        return [情绪安抚Agent, 认知重构Agent]
    elif "学业" in user_input or "学习" in user_input:
        return [行为指导Agent, 认知重构Agent]
    else:
        return [情绪安抚Agent, 行为指导Agent]
```

#### 4.2.2 Agent间协作流程

```text
用户输入 → 主Agent分析 → 协作Agent补充 → 综合响应生成
```

```python
def professional_matching(user_profile, issue_type, urgency):
    """
    专业人员匹配算法
    """
    professionals = get_available_professionals()
    
    # 匹配评分
    match_scores = []
    for prof in professionals:
        score = 0
        # 专业匹配度
        score += specialty_match_score(prof.specialties, issue_type)
        # 时间匹配度
        score += availability_match_score(prof.availability, urgency)
        # 历史效果
        score += historical_effectiveness_score(prof.id, user_profile.id)
        
        match_scores.append((prof, score))
    
    # 选择最佳匹配
    best_match = max(match_scores, key=lambda x: x[1])
    return best_match[0]
```
