HeartChat功能实现流程图

本文档展示了HeartChat项目各个核心功能的实现流程，包括云函数、API调用和数据流向之间的关系。

1. 用户登录与认证流程

flowchart TD A[用户点击登录按钮] --> B[调用 wx.login 获取 code] B --> C[调用 login 云函数] C --> D{用户是否存在?} D -->|是| E[获取用户信息] D -->|否| F[创建新用户] F --> E E --> G[返回用户信息和 openid] G --> H[保存用户信息到本地缓存] H --> I[跳转到首页] subgraph 云函数 login C1[接收 code] --> C2[调用微信 code2Session 接口] C2 --> C3[获取 openid] C3 --> C4[查询 users 集合] C4 --> C5[返回用户信息] end subgraph 数据库 DB1[(users 集合)] C4 <--> DB1 end

2. 聊天系统实现流程

flowchart TD A[用户发送消息] --> B[前端显示用户消息] B --> C[调用 chat 云函数] C --> D[保存用户消息到数据库] D --> E[调用智谱AI API] E --> F[生成AI回复] F --> G[保存AI回复到数据库] G --> H[返回AI回复和情绪分析结果] H --> I[前端显示AI回复] I --> J[更新情绪分析面板] subgraph 云函数 chat C1[接收消息和角色ID] --> C2[查询角色信息] C2 --> C3[构建提示词] C3 --> C4[调用智谱AI GLM-4-Flash API] C4 --> C5[处理AI回复] C5 --> C6[保存消息到数据库] C6 --> C7[返回结果] end subgraph 数据库 DB1[(roles 集合)] DB2[(chats 集合)] DB3[(messages 集合)] C2 <--> DB1 C6 <--> DB2 C6 <--> DB3 end subgraph 外部API API1[智谱AI GLM-4-Flash API] C4 <--> API1 end

3. 情感分析系统实现流程

flowchart TD A[用户发送消息] --> B[调用 analysis 云函数] B --> C[分析文本情感] C --> D[提取关键词] D --> E[关键词分类] E --> F[关键词情感关联] F --> G[保存情感分析结果] G --> H[返回分析结果] H --> I[前端显示情感分析结果] I --> J[更新情感面板] subgraph 云函数 analysis C1[接收文本] --> C2[调用智谱AI API] C2 --> C3[处理情感分析结果] C3 --> C4[提取关键词] C4 --> C5[关键词分类] C5 --> C6[关键词情感关联] C6 --> C7[保存结果到数据库] C7 --> C8[返回结果] end subgraph 数据库 DB1[(emotionRecords 集合)] DB2[(userInterests 集合)] C7 <--> DB1 C7 <--> DB2 end subgraph 外部API API1[智谱AI GLM-4-Flash API] API2[智谱AI Embedding-3 API] C2 <--> API1 C4 <--> API1 C5 <--> API1 C6 <--> API2 end
